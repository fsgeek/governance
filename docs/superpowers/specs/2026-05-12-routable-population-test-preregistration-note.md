# The routable-population test — does within-Rashomon disagreement *route* on the subprime tiers? — pre-registration

**Date:** 2026-05-12.
**Status:** Pre-registration. **No experimental code has been written at the time of this note.** Predictions and hit/miss criteria recorded before any code touches the data. Same discipline as the preceding pre-regs in this arc (`…disagreement-routing-…`, `…disagreement-geometry-…`, `…extension-admitted-band-…`).
**Origin.** The **extension-admitted-band** result (`2026-05-12-extension-admitted-band-result-note.md`) found that when the within-grade policy-constrained refinement ε-band is rebuilt with the 9 extension underwriting features *admitted* into the candidate set, the per-borrower disagreement `d_ext(x)` among the band members splits **perfectly by letter grade**: prime tiers (A1 R²_named 0.66, A5 0.42, B1 0.42 — `well_explained`) keep the disagreement legible in the documented 4 policy features; subprime tiers (C1 0.06, C2 0.11, C5 0.13, D4 0.15 — *not* `well_explained`) collapse it off the policy vocabulary onto the extension features. The disagreement-routing **deflation** (`2026-05-12-disagreement-routing-result-note.md` — per-case disagreement did *not* localize to where the consensus is unreliable; the high-disagreement tercile was if anything slightly *more* predictable / better-calibrated, because disagreement tracked where the within-tier residual feature is *active* = where the consensus is *better*) was thereby called **quality-stratified** — "the policy is blind here → route" is potentially real, but only on the subprime tiers. This note tests that directly. It is the routing→geometry→extension arc's "sharpest open thread" (work-state anchor §NEXT.1).
**The mechanism this test is built to detect.** On the subprime tiers the extension-admitted band averages, within ε of the best holdout AUC, (a) members whose tree splits on an extension feature (e.g. `inq_last_6mths`, `revol_util`) and (b) members near the tier base rate. Where the *truth* tracks the extension feature, the consensus `p̄(x)` (the mean over members) is **biased toward the base rate** while a good member is not — so the disagreement at `x` is large *and* the consensus at `x` is genuinely unreliable, unlike the #6-named-only case where large disagreement meant "the named residual feature is active here = the consensus is *better* here." If that mechanism is real, routing the borrowers whose disagreement the **policy vocabulary cannot explain** should make the #6 routing metrics fire on the subprime tiers. If it is not real — if even the residual disagreement fails to localize unreliability — the #6 deflation is robust, per-case disagreement-routing is dead, and the extension-admitted finding is about *observability*, not *triage*. Both are terminal, reportable outcomes; this is not a gateway to a test #N+1.
**Applying `project_pre_registration_pattern`:** prior pre-regs in this arc failed by assuming uniformity. This one (a) bets the *aggregate* outcome at ~40% (i.e. the modal outcome I'd actually bet on is heterogeneous — one tier routes, not three), (b) names the single tier I'd back (C1 — sharpest collapse), (c) makes "nothing routes anywhere" an explicitly pre-registered, non-disappointing result.
**Two design choices that were on the table and are now fixed (Tony's "dumb questions", 2026-05-12):** (i) the routed/kept comparison is run *both* ways — primary = the within-top-`d_ext`-tercile contrast (policy-blind half vs named-explained half), secondary = policy-blind vs the full complement — so the named-vs-not axis is isolated from the high-vs-low-disagreement axis; (ii) the protected-class-*footprint* check (does the policy-blind population cluster geographically) stays a **separate follow-up**, not folded into this verdict — but see §6 for the broadened proxy set it should consider and a non-load-bearing observation about the extension features themselves.
**Companion:** result note `docs/superpowers/specs/2026-05-12-routable-population-test-result-note.md` (after the run).

## 1. Question

On the subprime tiers where the extension-admitted band's disagreement `d_ext(x)` is *not* a legible function of the 4 documented policy features:

1. **De-dup robustness.** Does the quality-stratified split survive replacing the band's exact-tree-signature de-duplication with **de-duplication by used-feature-set** (the band-bloat fix — and the more principled notion of "distinct refinement": a refinement keyed on a distinct *set of features*, not a distinct set of thresholds)? I.e. does `R2_named_ext` stay < 0.3 on the subprime tiers with the de-duped band?
2. **Does the policy-blind disagreement route forward?** Define the **residual disagreement** `r(x) = d_ext(x) − d̂_named(x)`, where `d̂_named` is a depth-≤3 regression CART's prediction of `d_ext` from the 4 named features. Bucket the held-out quarter's borrowers by terciles of `r`. Do the #6 routing metrics — within-tercile member AUC vs a grade-size-aware null; calibration-gap–vs-disagreement Spearman; consensus Brier high-tercile minus low; operational ECE lift from routing out the high tercile — point the routing way on a majority of the subprime tiers, when keyed on `r`?
3. **Is "high disagreement" worth routing only when the policy can't name it?** Within the top `d_ext` tercile, split by `r` (above / below its median *within that tercile*) into a *policy-blind half* and a *named-explained half*. Is the consensus measurably worse (higher Brier, lower AUC, larger ECE) on the policy-blind half?
4. **Controls.** Does the *raw* `d_ext`-tercile bucketing route any better or worse than the residual-`r` one (does the residual subtraction help or just add the explainer's estimation noise)? Does the `d̂_named`-tercile bucketing (the *explained* part of the disagreement) **deflate**, replicating the #6 null?

## 2. Setup

- **Substrate / burst.** LC 36-month, Burst D (the load-bearing 2015H2 `dti` burst). Bands built on **V₁ = 2015Q3**; routing metrics forward-evaluated on **V₂ = 2015Q4** (band frozen); a V₁-in-sample arm reported for robustness. Flagged Cat-2-(pricing) grades in 2015Q3 read from `runs/pricing-lc-sweep-36mo-summary.json` (not hardcoded). Policy `policy/thin_demo_hmda.yaml`.
- **Extension-admitted band, de-duped by used-feature-set.** `wedge.refinement_set.build_refinement_band` over the candidate set = named ∪ extension (`{fico_range_low, dti, annual_inc, emp_length}` ∪ `{revol_util, open_acc, inq_last_6mths, delinq_2yrs, total_acc, mort_acc, loan_amnt, revol_bal, pub_rec}`, those present in LC accepted data), `monotonic_cst_map = {fico_range_low: −1, dti: +1}` (the thin-demo policy's directions on the *named* features only, sign-flipped to default-prediction; extension features unconstrained), ε = 0.02, depths {1, 2, 3}, leaf-mins {25, 50, 100}, seed 0, train/holdout 0.3 split. **Then**, from `band.members` (all combos within ε of the best holdout AUC), re-de-duplicate by the used-feature-set of the *train-split* tree — extractable exactly from the member's `tree_signature = (subset, ((feature_idx, threshold), …))` as `{subset[i] for (i, thr) in nodes}` (possibly empty for a stump that didn't split — the constant model, kept as one representative) — keeping the highest-`holdout_auc` representative per used-feature-set. Call these the *distinct-by-used-feature-set* members. Refit each on the full V₁ grade-G data, frozen. `d_ext(x)` = std (ddof=0) over the refit members' predicted-default probabilities; computed on V₁ grade-G (for fitting the explainer) and on V₂ grade-G (for the forward routing metrics).
- **Plurality (#6 definition, on the de-duped band).** A grade is *plural* iff ≥ 3 distinct-by-used-feature-set members **and** ≥ 2 distinct non-empty used-feature sets **and** median pairwise ranking-Spearman among the refit members' V₁ scores < 0.9.
- **Subprime set / primary aggregate.** The "subprime" grades = the flagged Cat-2 grades that were `well_explained = False` in the extension-admitted-band test (`R2_named_ext < 0.3` and `|ρ_top| < 0.3`): **C1, C2, C5, D4**. The **primary aggregate** = those of {C1, C2, C5, D4} that are *plural under the de-duped band on V₁*. On the (tree-signature-de-duped) extension-admitted result this was {C1, C2, C5} (D4 had median ρ 0.926 > 0.9); the de-duped-by-used-feature-set band may differ — the aggregate is re-derived, not hardcoded, and any change from {C1, C2, C5} is reported. Majority = strictly > (size)/2. Prime grades (A1, A5, B1) and any non-subprime flagged grade are reported per-grade for contrast but not in the aggregate.
- **Named-feature explainer & residual.** On V₁ grade-G: `tree_named = DecisionTreeRegressor(max_depth=3, min_samples_leaf=50, random_state=0).fit(X₁_named, d_ext_v1)`. `d̂_named` = `tree_named.predict(X_named)` on V₁ (in-sample arm) and V₂ (forward arm). `r = d_ext − d̂_named` on each. For the §1.1 / P5 R² numbers — comparable to the geometry and extension-admitted tests — `R2_named_ext` and `R2_all_ext` come from `wedge.routing.disagreement_explainer` (depth ≤ 3, leaf-min 50, 5-fold CV R²) on (`d_ext_v1`, named) and (`d_ext_v1`, named ∪ extension); the plain-`.fit` `tree_named` is used *only* for the point predictions `d̂_named`.
- **Bucketings.** Three, each computed on V₂ (forward, primary) and V₁ (in-sample, robustness):
  - **(i) `r`-terciles** — the *policy-blind* bucketing; the primary object of P1/P4.
  - **(ii) `d_ext`-terciles** — the raw-disagreement bucketing; control / baseline.
  - **(iii) `d̂_named`-terciles** — the *policy-explained* disagreement; negative control (should deflate, replicating #6).
  Terciles assigned by `wedge.routing.tercile_labels` (rank-based, equal-sized).
- **Routing metrics (`wedge.routing`, identical to the #6 routing test).** For each bucketing, per grade, on each arm: m1 `within_tercile_member_auc` (n_perm 500, p95 null); m2 `calibration_gap_vs_disagreement` (10 deciles of the bucketing variable); m3 `brier_by_tercile`; m4 `operational_lift` (route out the top tercile, 10-bin ECE); plus the `feature_space_outlierness` diagnostic; plus `grade_routing_verdict` → the booleans `m1_tercile_predictability_routing`, `m2_calibration_gap_routing`, `m3_brier_routing`, `m4_operational_lift_nonneg`.
- **Within-top-`d_ext`-tercile contrast (§1.3 / P1b).** Among the borrowers in the top tercile of `d_ext` (on each arm): `policy_blind_half` = those with `r` above its median within that tercile; `named_explained_half` = the rest. Report, per half: n; consensus Brier `mean((p̄ − y)²)`; consensus AUC for realized default (None if degenerate); 10-bin ECE; **and** the minimum over band members of that member's Brier (does a *good model exist* in the policy-blind half — i.e. is the consensus's failure there a failure of *averaging*, not of the class?). The pre-registered routing-relevant pattern: consensus Brier(policy_blind_half) > consensus Brier(named_explained_half) **and** min-member Brier(policy_blind_half) < consensus Brier(policy_blind_half) by more than it is in the named-explained half (a good member is being averaged away). Secondary: the same numbers for `policy_blind_half` vs the *full complement* of the tier.

## 3. Metrics & verdict logic (all pre-registered, all reported)

Aggregate over the primary set P = {plural subprime grades on V₁} (size n; majority = strictly > n/2). All on the V₂ forward arm unless noted; V₁ in-sample is the robustness arm.

- **P5 — de-dup robustness:** **P5-hit** iff `R2_named_ext < 0.3` on > maj of P (the quality-stratified split survives the used-feature-set de-dup on the subprime tiers). Also report, for the prime grades, whether `R2_named_ext ≥ 0.3` still holds (the split's other half).
- **P1a — policy-blind disagreement routes (terciles of `r`):** for the `r`-bucketing, the load-bearing direction is "the consensus is *worse* where the policy is blind" — i.e. m3 (`brier_by_tercile.high_minus_low > 0`) and m4 (`operational_lift.lift ≥ 0`). **P1a-hit** iff (`m3_brier_routing` on > maj of P) **AND** (`m4_operational_lift_nonneg` on > maj of P). m1 and m2 are reported for the `r`-bucketing but **not gating** — their expected sign is ambiguous under the mechanism (if the extension-feature members predict realized default well on the policy-blind tercile, mean-member-AUC there is *high*, which fails the #6-style m1 criterion while being *consistent with* the routing story — a good model exists, the consensus just isn't it; this asymmetry is itself recorded). The #6-style m1∧m2∧m3 triple is also reported for the `r`-bucketing as the cross-test-comparable number.
- **P1b — high disagreement routes only when the policy can't name it:** **P1b-hit** iff, on > maj of P, consensus Brier(policy_blind_half) > consensus Brier(named_explained_half) (within the top `d_ext` tercile). Report the min-member-Brier gap pattern alongside (descriptive, not gating).
- **P1 (headline) = P1a AND P1b.** "Within-Rashomon disagreement *does* route on the subprime tiers — but only the part the policy vocabulary cannot explain, and the routable population is identifiable from the band + the named-feature explainer alone (no held-out outcomes)."
- **P2 — the explained part deflates (negative control, terciles of `d̂_named`):** **P2-hit** iff the `d̂_named`-bucketing does **not** show `m3_brier_routing` on > maj of P (and ideally m1/m2/m3 falsified-majority, matching #6). If P2 *misses* — the explained part routes too — the residual decomposition is confounded (disagreement routes or doesn't regardless of how it's split) and P1, if it also hit, must be reported with that caveat front and centre.
- **P4 — residual beats raw (control, terciles of `d_ext` vs `r`):** **P4-hit** iff the `r`-bucketing's #6-style m1∧m2∧m3 routing-yes count ≥ the raw `d_ext`-bucketing's, on the aggregate (the residual is at least as clean a router as the raw disagreement). A miss here ("raw routes as well or better") is informative: the named-explainer subtraction adds noise; report the simpler bucketing as the operative one.
- **Per-grade verdict** (descriptive, no aggregate gate): for each grade in P and each prime grade, the four routing booleans on each of the three bucketings, on each arm; `R2_named_ext`, `R2_all_ext`, root feature; plurality; band size before/after the used-feature-set de-dup.

**Headline forms.** "On the subprime LC 2015H2 tiers {…}, the within-Rashomon refinement-band disagreement, restricted to the part the 4-feature documented policy cannot explain, [does / does not] localize the borrowers on which the band consensus is least reliable forward ([k]/[n] tiers by Brier, [k]/[n] by ECE lift), [and the explained part of the disagreement [does not / does] route — the decomposition is [clean / confounded]]; the routable population [is identifiable from the band + the named-feature explainer without held-out outcomes / does not exist]; [per-case disagreement-routing is partially un-deflated on these tiers on a 'the documented policy is silent about the contested feature' footing (fair-lending-adjacent) / per-case disagreement-routing is dead even with the wider feature view — the extension-admitted finding is about observability, not triage]." Also: "the quality-stratified split [survives / does not survive] the used-feature-set de-duplication (R²_named on the subprime tiers [·]; on the prime tiers [·])."

## 4. Pre-registered predictions

> **P5 — lean yes, ~80%.** The split was perfect (all 7 flagged grades) on the tree-signature-de-duped band; de-duping by used-feature-set *shrinks* the band but doesn't change which features the surviving members key on, so the named explainer's grip should track. *Hedge:* the de-duped band is much smaller (≤ ~160 members vs ≤ ~2900); a smaller, less-redundant band could push `R2_named_ext` around by ±0.05–0.10 — if a subprime grade lands at 0.28–0.34 it's noise around the line, report it as such.
>
> **P1 — genuinely uncertain, ~40% (the fun one — real coin flip).** The mechanism (consensus biased toward base rate where the truth tracks an extension feature) is plausible *but* the #6-killing reasoning — "disagreement marks where there's signal, which is where prediction is *easy*, not hard" — could survive the residual subtraction: the residual `r` could still concentrate where an extension feature is *active and predictive*, in which case realized default there is predictable (just not by the consensus), m3 (consensus Brier) might still fire but m4 (ECE lift) might not (routing out a *predictable-by-someone* tercile needn't tighten the *consensus's* calibration on the rest), and P1a would miss on the m4 leg. I will not pretend a prior beyond "near a coin flip, leaning miss on the *aggregate*."
>
> **Per-tier heterogeneity (the explicit anti-uniformity prediction).** Modal outcome I'd actually bet on: **exactly one of {C1, C2, C5(, D4)} routes on P1a**, and I'd back **C1** — it had the sharpest collapse (`R2_named_ext` 0.06, the lowest), so if "the disagreement keys on un-named structure → the consensus is biased there" is real, it should be cleanest where the un-named structure most dominates. C5 (`d_ext` driver `dti`, a *named* feature — its collapse is more about the band being filled with near-base-rate trees than about an extension feature carrying the residual) I'd back to *deflate*. C2 a coin flip. **I will be surprised if all of P routes the same way** (either all-route or all-deflate); a clean split is the bet, which would make the *aggregate* P1a a miss but leave a real "C1 is routable, the others aren't" finding — which is itself a publishable nuance (the routable subset is narrower than "the subprime tiers").
>
> **P2 (control deflates) — ~70%.** The explained part `d̂_named` is, by construction, `d_ext` filtered through the named features — i.e. exactly the #6-named-only-band situation, which deflated. It *should* deflate again. If it doesn't (~30%), the most likely reason is that `d̂_named` and `d_ext` are highly correlated on these tiers (the residual is small) so the bucketings barely differ — in which case P1, if it hit, is on thin ice. The single most important number to look at in the result is **how different the three bucketings actually are** (rank correlation between `r`-terciles and `d_ext`-terciles, and between `d̂_named`-terciles and `d_ext`-terciles); if `r`-terciles ≈ `d_ext`-terciles the residual decomposition did nothing.
>
> **P4 (residual ≥ raw) — ~55%.** Weak lean: the residual *should* be the cleaner object in principle, but `d̂_named` on the subprime tiers explains so little (R² ~0.06–0.15) that `r ≈ d_ext` minus a near-constant — i.e. the subtraction may be nearly a no-op and P4 a near-tie. If `R2_named_ext` is genuinely ~0.06, the residual subtraction is basically removing the explainer's mean, and P4 will land "tie".
>
> **Cross-cutting most-likely miss.** Consistent with the geometry and extension-admitted tests (where I *under*-predicted legibility / decisiveness on the prime side), my likeliest specific miss: **m3 fires broadly but m4 doesn't** — the consensus *is* worse on the high-`r` tercile (it's biased there) but routing that tercile out *doesn't* tighten the kept population's ECE (because the kept population was already fine, or because removing the high-`r` cases changes the quantile bins in a way that washes the lift out). That would be P1a-miss-on-the-m4-leg with P1b possibly still hitting — i.e. "the consensus is identifiably worse where the policy is blind, but you don't gain calibration by routing those cases out." Reportable, and a sharper statement than a flat deflation: the routing *signal* is real, the routing *action* doesn't pay.

## 5. Why this is the right test now

- It is the one open thread that can *un-deflate* a piece of the disagreement-routing claim — and it does so on the footing the extension-admitted result handed up: not "the models are confused about this borrower" (the per-case-uncertainty version, tested and deflated) but "the documented policy is *silent* about the feature the admissible refinements are fighting over, *and that silence is concentrated on the tiers where ECOA/Reg-B disparate impact concentrates*." A hit feeds the regulator-doc spine's §A.3↔§B "weave tighter once P4 lands" placeholder and the fair-lending add (ii) with concrete content; a miss closes the per-case-routing question cleanly and tells the regulator doc to scope §B to *observability* (the policy's vocabulary is inadequate to the indeterminacy here — that's the supervisory fact) rather than triage.
- It ships the **band-bloat fix** (de-dup by used-feature-set) as a side effect — the more principled "distinct refinement", the runtime fix for the O(members²) Spearman, and a robustness re-check on the quality-stratified split, all in one pass (kills the work-state menu's "used-feature-set-de-dup re-run" item).
- It reuses `wedge/refinement_set.py` and `wedge/routing.py` wholesale (the same metric machinery as the #6 routing test, so the numbers are directly comparable) — no new `wedge/` code, no new unit tests; the new code is one script (`scripts/routable_population_test.py`), one results JSON, two notes. The data load is the ~5-named-+-9-extension-column LC read (`usecols`), no FM.

## 6. Out of scope here — the protected-class-footprint follow-up

Deliberately *not* in this test (it is already four-prediction-shaped), but recorded so it is not lost:

- **The question.** If P1 hits — there is a routable "policy-blind" population on the subprime tiers — does that population have a **disparate footprint**? I.e. is "the documented policy is silent about the contested feature" not just a model-risk fact but a fair-lending one?
- **The proxy hazard (Tony, 2026-05-12).** Geographic/identifier proxies are richer than 3-digit `zip_code`: `addr_state`; `earliest_cr_line` (an age proxy — age is itself ECOA-protected); `emp_title` (free-text occupation); `home_ownership`. The general hazard: a field that looks like an opaque identifier can encode geography-at-issuance which then proxies for protected status — historical SSN area numbers (pre-2011 randomization; e.g. 540 ⇒ Oregon), telephone exchanges in single-carrier areas, etc. The footprint test should look at the *set* of these in the LC data, not just zip3.
- **The clean substrate.** HMDA carries *reported* race/ethnicity/sex (no BISG proxying needed) — it is a different pipeline but the right place to ask the disparate-impact question without proxy inference. The LC zip3/state/`emp_title` version is a screening pass, not the answer.
- **A non-load-bearing observation, recorded now.** The 9 extension features the subprime disagreement collapses onto include wealth/credit-depth variables (`revol_bal`, `loan_amnt`, `mort_acc`, `total_acc`) which are themselves correlated with protected-class status — so "the policy is blind here" already carries a faint disparate-impact signal *before* any geographic test. This is *not* part of the §3 verdict; it is a flag for the follow-up's framing.
