# Extension-admitted refinement bands and the locus of within-Rashomon disagreement — pre-registration

**Date:** 2026-05-12.
**Status:** Pre-registration. **No experimental code has been written at the time of this note.** Predictions and hit/miss criteria recorded before any code touches the data. Same discipline as the preceding pre-regs in this arc.
**Origin.** The **disagreement-geometry** result (`2026-05-12-disagreement-geometry-result-note.md`) confirmed that `d(x)` — the per-borrower disagreement among the #6 policy-constrained refinement-band members — is a legible, low-dimensional, *tier-idiosyncratic* function of the policy-named features (CV R² 0.46–0.86 on the 5 plural Burst-D grades; driver `dti` on A5/D4, `annual_inc` on B1/C1, `fico` on C5; concentrated in the tails). But its **P4 (rescue) branch was structurally vacuous**: the #6 bands are built over the policy-named features `{fico_range_low, dti, annual_inc, emp_length}` *only*, so `d` is *by construction* a pure function of them — the 9 extension underwriting features `{revol_util, open_acc, inq_last_6mths, delinq_2yrs, total_acc, mort_acc, loan_amnt, revol_bal, pub_rec}` add ΔR² ≈ 0 trivially. The geometry note (§6, top priority) and the work-state anchor both name the repair: re-run the #6 band construction with the extension features **admitted into the candidate set**, recompute `d`, and ask whether the disagreement *stops being explainable from the policy vocabulary alone*. This note is that test. It is the one branch from the routing→geometry arc that could **partially un-deflate** the disagreement-routing result — on a different and arguably stronger basis than the per-case-uncertainty version that was tested (`2026-05-12-disagreement-routing-result-note.md`): not "the models are confused about this borrower" but "the documented policy is *silent* about the feature the admissible refinements are fighting over" — which is **ECOA/Reg-B disparate-impact adjacent** (cf. the Sonnet/Hamut'ay add (ii) to the regulator-doc spine) and feeds the §A.3↔§B "weave tighter once P4 lands" placeholder.
**Applying `project_pre_registration_pattern`:** the geometry P4 failed by *malformation* (a prediction applied to a construction that couldn't satisfy it), not by the usual assume-uniformity failure. This note (a) uses the construction that *can* satisfy it, and (b) explicitly models tier heterogeneity — the verdict is an aggregate over the plural grades, but the per-grade prediction below differs by tier and a same-direction sweep across all 5 would itself be a surprise.
**Companion:** result note `docs/superpowers/specs/2026-05-12-extension-admitted-band-result-note.md` (after the run).

## 1. Question

When the within-grade policy-constrained refinement ε-band is rebuilt with the **extension underwriting features admitted** into the candidate feature set (the policy's monotonicity directions still enforced on the *named* features; the extension features unconstrained, because the documented policy says nothing about them):

1. **Do the band's distinct members actually use the extension features** — i.e. does admitting them change the band at all, or is the policy vocabulary AUC-sufficient within these tiers at ε = 0.02?
2. **Is the recomputed disagreement `d_ext(x)` still well-explained by the policy-*named* features alone** — or has the locus of disagreement moved to structure the policy vocabulary cannot name?
3. **How much does the named-feature explainer's grip on `d` *drop*** relative to the #6 (named-only) band — per tier?
4. **Does admitting the extension features change which grades are *plural*** — was the policy constraint suppressing (or inflating) the indeterminacy?

## 2. Setup

- **Substrate / burst:** LC 36-month, Burst D (the load-bearing 2015H2 `dti` burst). Bands built on **V₁ = 2015Q3**, robustness arm on **V₂ = 2015Q4** (band frozen, `d_ext` recomputed). Flagged Cat-2-(pricing) grades in 2015Q3 read from `runs/pricing-lc-sweep-36mo-summary.json` (not hardcoded). Policy `policy/thin_demo_hmda.yaml`.
- **Extended band construction.** `wedge.refinement_set.build_refinement_band` over the candidate set = **named ∪ extension** (`avail_all`: `{fico_range_low, dti, annual_inc, emp_length}` ∪ `{revol_util, open_acc, inq_last_6mths, delinq_2yrs, total_acc, mort_acc, loan_amnt, revol_bal, pub_rec}` — those present in LC accepted data). `monotonic_cst_map` = the thin-demo policy's directions on the named features only, sign-flipped to default-prediction (`{fico_range_low: −1, dti: +1}`); extension features get no entry → unconstrained. Same ε = 0.02, depths {1, 2, 3}, leaf-mins {25, 50, 100}, seed 0, train/holdout 0.3 split, de-dup by tree signature. (NB this enlarges the subset search from 2⁴−1 = 15 to 2¹³−1 = 8191 combos × 9 hyperparameter cells per grade — fine on LC sub-grade-sized data, ≤ ~74k tree fits per grade, depth ≤ 3.) Distinct members refit on the full V₁ grade-G data, each on its own used subset, frozen. `d_ext(x)` = std over the distinct members of predicted-default probability on x.
- **Plurality (same #6 definition):** a grade is *plural* iff ≥ 3 distinct members **and** ≥ 2 distinct *used*-feature sets (features the fitted trees actually split on) **and** median pairwise ranking-Spearman among members < 0.9. The "headline plural set" for the verdict = the intersection of {plural under the extended band on V₁} and {the #6 plural set: A5, B1, C1, C5, D4}; the full extended-plural set and the #6-only and extended-only differences are all reported.
- **Disagreement explainer (identical to the geometry test, so R² is comparable):** for each headline-plural grade G, a regression CART, `max_depth ≤ 3`, `min_samples_leaf = 50`, 5-fold-CV R², predicting `d_ext` from (a) the **named** features, (b) the **named + extension** features. Record the root-split feature, top-3 importances, and the partial-dependence shape of `d_ext` on the named-explainer's root feature (`tails` / `threshold` / `monotone` / `flat`, by the same decile-mean rule). Univariate Spearman ρ(f, `d_ext`) for each feature; threshold-proximity Spearman of `|dti − 43|` and `|fico_range_low − 620|` with `d_ext`.
- **Extension usage diagnostic.** Per grade: the union over distinct members of `used_feature_set(tree) ∩ extension features`; the number / fraction of distinct members whose tree splits on ≥ 1 extension feature; the per-feature member-usage count.
- **Comparison to #6 (the named-only band).** Per headline-plural grade: `R2_named_ext` (this test) vs `R2_named_#6` (the geometry result: A5 0.46, B1 0.57, C1 0.69, C5 0.46, D4 0.60); `ΔR2_band = R2_named_ext − R2_named_#6` (negative = the named explainer lost grip when the band could see more); whether the root-split feature of the named explainer changed.

## 3. Metrics & verdict logic (all pre-registered, all reported)

Per headline-plural grade G, on V₁ (primary) and V₂ (robustness):
- `R2_named_ext`, `R2_all_ext`, `ΔR2_ext = R2_all_ext − R2_named_ext`.
- `uses_extension` = (≥ 1 distinct member's fitted tree splits on ≥ 1 extension feature); `frac_members_using_extension`.
- `top_named` = root-split feature of the named-only explainer of `d_ext`; `|ρ_top|`; `shape`.
- `ΔR2_band` (vs the #6 named-only band's named-explainer R² on `d`).
- `prox_ρ` = max(|Spearman(|dti−43|, `d_ext`)|, |Spearman(|fico−620|, `d_ext`)|).

Aggregate over the headline-plural grades (n; majority = strictly > n/2). "well-explained-by-named" = `R2_named_ext ≥ 0.3` OR `|ρ_top| ≥ 0.3`.

- **A — "does admitting the extension features change the band":** **A-hit** iff `uses_extension` on > maj of grades. *If A misses* → the policy vocabulary is AUC-sufficient within these tiers at ε = 0.02; the extension features don't enter a competitive refinement; the disagreement-routing deflation is *reinforced*, not rescued, and P4 below is moot in the "blindness" direction (it will mechanically miss because `d_ext ≈ d`). A-miss is a clean, reportable result, not a null.
- **P4 (the rescue, headline) — "the disagreement keys on un-named structure":** **P4-strict-hit** iff `R2_named_ext < 0.3` on > maj **AND** `ΔR2_ext ≥ 0.1` on > maj — the named features *don't* explain `d_ext` but adding the extension features *does*. **P4-soft-hit** iff `ΔR2_band ≤ −0.15` on > maj — the named explainer's grip dropped by ≥ 15 R²-points relative to the #6 band even if it didn't fall below 0.3 absolute (the "the disagreement *partly* moved to un-named structure" reading; anticipates the geometry result's "modest, not all-or-nothing" pattern). The strict version is the headline; the soft version is the "partly" fallback.
- **P5 — "plurality side-effect":** **P5-hit** iff the extended band makes ≥ 1 #6-non-plural flagged grade (in Burst D: A1, C2) plural, **or** makes ≥ 1 #6-plural grade non-plural. Report the direction and which grades.

**Headline forms:** "Admitting the 9 extension underwriting features into the policy-constrained refinement band [does / does not] change which refinements are admissible within these tiers ([k]/[n] grades pick up an extension-feature member); the disagreement among them [remains a legible function of the policy-named features / moves to structure the policy vocabulary cannot name (R²_named drops to [·], the extension features recover [·])]; the disagreement-routing deflation [stands / is partially un-deflated on a 'the documented policy is blind here' basis — which is fair-lending-adjacent]."

## 4. Pre-registered predictions

> **A — lean yes (~60%).** The extension features `{revol_util, inq_last_6mths, delinq_2yrs, ...}` are exactly the credit-risk variables a 4-feature thin-demo policy omits and which carry independent signal in LC ([[project_codification_infrastructure]] — "the filter only had teeth because we added 9 extra features"); ε = 0.02 is generous enough for a marginally-useful feature to land within the band. *Hedge:* within a single sub-grade, `sub_grade` has already absorbed the bulk of the FICO/DTI/income signal, so the within-tier residual the extension features could grab is thin — I expect A-grades (A5, the non-plural A1) most likely to stay all-named-only, lower grades more likely to admit an extension member.
>
> **P4-strict — genuinely uncertain, ~35–40% (the fun one — no strong prior).** If it fires, a piece of the disagreement-routing claim is rebuilt on a "the policy is silent about the contested feature" footing (a model-risk *and* fair-lending fact, not a per-case heuristic), and the regulator-doc §B gets a concrete "and here is where it's worth routing" hook. If it misses, the residual-tracking reading *hardens*: even with a wider feature view the band's disagreement stays named-legible — the policy vocabulary, though thin, is the vocabulary the indeterminacy lives in. I will not pretend to a prior beyond "near a coin flip, leaning miss."
>
> **P4-soft (ΔR2_band ≤ −0.15 on > maj) — ~50%.** I expect *some* erosion of the named explainer's grip on most grades once extension-feature members enter (they do *something* to `d`), but whether the average drop crosses 15 R²-points is itself a coin flip.
>
> **P5 — ~40% that something flips; lean toward "C2 becomes plural."** C2 was non-trivial-but-not-plural in #6 — close to the plurality line; one extra used-feature set from an extension feature could push it over. Less likely: a #6-plural grade loses plurality (would need the extension members to all collapse to one used set, unlikely).
>
> **Per-tier heterogeneity (the explicit anti-uniformity prediction).** I expect **B1 most likely to show P4-ish behaviour** — its #6 disagreement-driver was *income*, and `revol_bal` / `loan_amnt` / `mort_acc` are borrower-scale features that could carry disagreement income alone can't name; **A5 and D4 least likely** (clean `dti` drivers, A/D-grade, thin residual). I will be **surprised if all 5 plural grades move together** in either direction; a split (e.g. P4 fires on B1/C1, misses on A5/C5/D4) is the modal outcome I'd bet on, which would make the *aggregate* P4-strict miss but leave a real tier-level finding.
>
> **Cross-cutting most-likely miss.** Consistent with the geometry result (where I *under*-predicted legibility), my likeliest miss is: A hits *more* than I expect (more extension features enter the bands) **but** `d_ext` stays named-explained anyway — the extension-feature members turn out near-redundant with named-feature members in the disagreement-relevant region (the tier interior, where everyone ≈ base rate, is where the extension features would have to differ to move `d`, and that's exactly where there's no residual for them to grab). I.e. "P4 misses *despite* A hitting" — the "structured, more features visible, still named-legible" reading. If that's what happens, it's a stronger statement of the deflation than a vacuous miss.

## 5. Why this is the right test now

The disagreement-routing result is deflated; the geometry result confirmed the residual-tracking reading but left its rescue branch structurally untested. This test is the *one* construction from the arc that can either (a) un-deflate a piece of routing on a regulator-relevant basis (policy blindness → route-to-human, fair-lending-adjacent), or (b) close the question by hardening the deflation ("even with a wider feature view, the indeterminacy is in the policy vocabulary, or the policy vocabulary is AUC-sufficient and the indeterminacy is small"). Every branch — A-miss, P4-strict-hit, P4-soft-only, P4-miss-despite-A — is a clean, reportable claim. It's cheap (LC 2015Q3/2015Q4 only, no FM load; ≤ ~74k shallow tree fits per grade). It directly feeds the near-term deliverable: the regulator-doc spine's §A.3↔§B bridge currently carries "weave tighter once the P4 test lands" as a placeholder, and the fair-lending footnote (Sonnet/Hamut'ay add (ii)) is conditional on this. And it repairs a self-caught pre-registration flaw with the experiment that can actually answer it.

## 6. Followups (not part of this pre-registration)

- **If P4-strict (or P4-soft) fires:** build the policy-blindness routing test directly — route the high-`d_ext` borrowers whose disagreement is *unexplained by the named features*, re-run the disagreement-routing metrics on that population; and check whether that population correlates with a geographic / protected-class proxy (the disparate-impact follow-up — converts "the policy is blind here" into "and the blindness has a disparate footprint" or rules it out).
- **If A misses (band stays all-named):** report it as the codification-infrastructure counter-instance — within these tiers the thin demo's 4 features are predictively, not just legally, sufficient at ε = 0.02; note that a *richer* policy (FNMA Selling Guide, #11) would change the candidate set and might re-open it.
- **FM variant:** rebuild FM's thin-demo bands with FM's extension columns (`ltv` etc.) admitted, repeat — confirms the mechanism's substrate-(in)dependence.
- Either way: `wedge.rashomon.select_diverse_members` on the extended band → the small presentable set, annotated with the tier disagreement statistic *and* the feature(s) (named or extension) the members fight over — the deliverable form.
